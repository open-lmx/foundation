# Asset Theory

Open-LMX treats every meaningful entity in an AI-native system as an asset.

An asset is any identifiable thing that can be governed, evaluated, related, versioned, trusted, used, owned, or retired.

## Asset Examples

- Human
- Agent
- Organization
- Team
- Role
- Skill
- Tool
- Model
- Dataset
- Memory
- Prompt
- Workflow
- Task
- Decision
- Policy
- Rule
- Control
- Objective
- Constraint
- Service
- Application
- Environment
- Infrastructure
- Artifact
- Document
- CreativeWork

## Base Asset Requirements

Every asset must declare:

- Identity
- Type
- Name
- Owner
- Version
- Status
- Lifecycle
- Relationships
- Policy
- Provenance
- Trust signals
- Audit metadata

## Asset Rule

No governed system may depend on an unidentified asset.
